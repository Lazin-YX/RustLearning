# 内存操作
从内存的角度, 任何变量都由如下参数组成:  
1. 一个usize类型的数值表示变量的首地址  
2. 变量类型占用的内存块大小  
3. 变量类型内存字节对齐的基数  
4. 变量类型的成员内存顺序  

与C不同, Rust中, 变量类型的成员内存顺序及布局完全由编译器控制, 不能由代码决定, 这有助于编译优化  
与C相同, Rust也能够直接将内存块转换成某一类型变量的功能, 该功能被限制为unsafe代码  

标准库内存相关模块主要包含以下内容:  
1. 编译器提供的内存操作固有函数  
2. 内存块与类型系统的结合点: 裸指针(`*const T`, `*mut T`)  
3. 裸指针的封装结构: `NonNull<T>`, `Unique<T>`  
4. 未初始化内存块的处理: `MaybeUninit<T>`, `ManuallyDrop<T>`  
5. 堆内存的申请与释放
